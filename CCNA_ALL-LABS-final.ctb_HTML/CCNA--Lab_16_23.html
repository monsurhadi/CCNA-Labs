Lab No : 16 :: VTP (VLAN Trunking Protocol)

VTP is a Cisco proprietary protocol used to synchronize VLAN information across switches in a network.

Modes of VTP

  1. 1.

    1. Server Mode (Default Mode)

  2. Can create VLANs
  3. Can modify VLANs
  4. Can delete VLANs
  5. Sends VTP advertisements to other switches
  6. Stores VLAN information in vlan.dat (flash memory)

  1. 2.

    2. Client Mode

  2. ❌ Cannot create VLANs
  3. ❌ Cannot modify VLANs
  4. ❌ Cannot delete VLANs
  5. ✔ Only receives VLAN updates from VTP server
  6. ✔ Automatically updates VLAN database from server

Example:

If Switch S2 is a client → it accepts VTP updates and updates its VLAN database automatically.

  1. 3.

    3. Transparent Mode

  2. Does NOT participate in VTP database synchronization
  3. Can create/modify/delete VLANs locally only
  4. Forwards VTP advertisements to other switches
  5. Does NOT update its VLAN database from VTP messages

Example:

Switch S3 will forward VTP advertisements but will not update its VLAN database.

VTP Advertisement Behavior

  • Sent from Server switches
  • Forwarded by Transparent switches
  • Received and applied by Client switches
  • Uses VTP revision number to determine updates
